MongoDB · Express · React · Node · TypeScript

1. Project Overview
A Kanban-style board for tracking job applications through a customisable pipeline. Users can move applications between stages, log interviews, attach notes, set reminders, and view analytics on their search activity.
Application statuses
Status
Description
wishlist
Not yet applied. Starting state for tracking a job before committing.
applied
Application submitted. Backend auto-sets appliedAt on transition.
interviewing
At least one interview scheduled. Auto-set by the create interview controller.
offer
Received an offer. Terminal positive state.
rejected
Rejected at any stage. Terminal negative state.
ghosted
No response after follow-ups. Distinct from rejected for analytics purposes.

8. Key TypeScript Patterns
Deriving types from constants with as const
Use as const instead of : readonly string[]. Without it, typeof ARRAY[number] resolves to string, making Record<Type, string> effectively Record<string, string> — TypeScript will not warn about missing keys.
// Wrong — loses literal types
const APPLICATION_STATUSES: readonly string[] = [...]
 
// Correct — preserves literal union
const APPLICATION_STATUSES = ['wishlist', 'applied', ...] as const
type ApplicationStatus = typeof APPLICATION_STATUSES[number]
// => 'wishlist' | 'applied' | 'interviewing' | 'offer' | 'rejected' | 'ghosted'

Zod schemas as the single source of truth
Define the schema once, derive the TypeScript type from it. Never write the type by hand — it will drift from the schema.
export const createApplicationSchema = z.object({
  company: z.string().min(1),
  role:    z.string().min(1),
  status:  z.enum(APPLICATION_STATUSES),
})
 
export type CreateApplicationInput = z.infer<typeof createApplicationSchema>
// TypeScript type is always in sync with the validator

Mongoose with TypeScript
import { Application } from '@job-tracker/shared'
import { Document, Model } from 'mongoose'
 
type ApplicationDocument = Application & Document
type ApplicationModel   = Model<ApplicationDocument>
 
// Use the shared interface — never redefine fields in the model file

Optimistic update pattern (TanStack Query)
const mutation = useMutation({
  mutationFn: (newStatus) => api.updateStatus(id, newStatus),
  onMutate: async (newStatus) => {
    await queryClient.cancelQueries({ queryKey: ['applications'] })
    const snapshot = queryClient.getQueryData(['applications'])
    queryClient.setQueryData(['applications'], (old) => (
      old.map(app => app._id === id ? { ...app, status: newStatus } : app)
    ))
    return { snapshot }
  },
  onError: (err, vars, ctx) => {
    queryClient.setQueryData(['applications'], ctx.snapshot)
  },
  onSettled: () => {
    queryClient.invalidateQueries({ queryKey: ['applications'] })
  },
})

Common first-run issues
Error
Fix
Cannot find module '@job-tracker/shared'
Run npm install from the root, not from inside a package folder.
MongooseServerSelectionError
Your Atlas cluster IP whitelist does not include your current IP. Add it in Atlas → Network Access.
Port 3001 already in use
Change PORT in packages/server/.env to 3002 or kill the process using port 3001.
CORS errors in browser
Check that the Vite proxy is configured in vite.config.ts and that the server is running on port 3001.
